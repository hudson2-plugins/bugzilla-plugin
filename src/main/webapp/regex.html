This is the regex used to match bug IDs in changeset comments. After a match is found, first the entire match, then every grouping within the match is parsed as an integer. The first successful parse is used as the bug ID.
To match any number, use: [0-9]+
The bugzilla style of matching bug + number would be: bug ([0-9]+) Note that the parens in this example are used to identify the bug number.

Here are two examples from the hudson community:

Example 1:
(?<=[([Bb]ug)([Ii]ssue)(SCR)(scr)]:?\s?#?)\d+
The above regex example matches only the numbers in strings like the following:
Bug #12345
Issue: 12345
SCR 12345
Bug 12345
bug 12345
bug #12345
scr 12345
scr12345
scr 123456
The regex does NOT match just any number. The number must be preceded by something like "bug", "issue" or "SCR".

Example 2:
(?<=[Bb]ug [\d, ]{0,64})(\d+)
It will match and link multiple bug IDs for example:
Bug 12
Bug 12,30
Bug 12 30 54
Bug 12, 30, 54, 81
